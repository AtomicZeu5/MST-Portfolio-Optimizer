Theoretical Foundation

Graph Theory in Finance

Asset correlations are represented as weighted edges in an undirected graph, where:

  • Vertices (V) represent individual securities
  • Edges (E) represent correlation relationships
  • Edge weights correspond to transformed correlation coefficients
Distance Transformation:
dij = √(2(1-ρij))

MST Algorithms

Kruskal's Algorithm

Approach: Edge-based, greedy selection

Time Complexity: O(E log E)

Data Structure: Union-Find

Best for: Sparse graphs

Prim's Algorithm

Approach: Vertex-based, growing tree

Time Complexity: O(E log V)

Data Structure: Priority Queue

Best for: Dense graphs

Network Centrality Measures

Degree Centrality

Number of direct connections to other assets

Betweenness Centrality

Measure of asset's role as intermediary in network

Eigenvector Centrality

Importance based on connections to other important assets

Algorithm Implementation

Data Structures Used

Graph Representation

Adjacency matrix and edge list for correlation data

Space: O(V²)

Union-Find (Disjoint Set)

Path compression and union by rank optimization

Time: O(α(n)) amortized

Priority Queue

Min-heap for Prim's algorithm edge selection

Operations: O(log V)

Sorting Algorithm

Edge sorting for Kruskal's algorithm

Time: O(E log E)

Kruskal's Algorithm Pseudocode

function kruskalsAlgorithm(graph):
    mst = []
    unionFind = UnionFind(graph.vertices)
    
    // Sort edges by weight (distance)
    edges = sortByWeight(graph.edges)
    
    for edge in edges:
        u, v, weight = edge
        
        // Check if adding edge creates cycle
        if unionFind.find(u) != unionFind.find(v):
            mst.append(edge)
            unionFind.union(u, v)
            
            // Stop when MST is complete
            if len(mst) == graph.vertices - 1:
                break
    
    return mst

Conclusion & Applications

Research Summary

This project successfully demonstrates the application of advanced data structures and algorithms in financial portfolio optimization. The MST-based approach provides valuable insights into asset correlation structures and offers a systematic framework for portfolio construction.

Technical Contributions

  • Efficient implementation of Kruskal's algorithm with Union-Find optimization
  • Network analysis using centrality measures for weight allocation
  • Comparative analysis of different MST-based optimization strategies
  • Integration of graph theory concepts with modern portfolio theory

Practical Applications

Risk Management

MST visualization helps identify concentration risks and correlation clusters for better diversification strategies.

Asset Selection

Centrality measures guide systematic asset screening and identification of redundant positions.

Dynamic Rebalancing

Network structure analysis enables tactical allocation adjustments based on evolving correlation patterns.

Systemic Risk Assessment

Betweenness centrality identifies systemically important assets that could propagate market shocks.

Limitations & Future Work

Future Research Directions
